El modelo de cajas: la base de todo diseño web

Por Ana Flores · 10 de mayo, 2025 · 5 min de lectura

Antes de escribir una sola línea de CSS, todo desarrollador web necesita entender un concepto fundamental: el modelo de cajas. Este modelo define cómo se calcula el tamaño y el espacio de cada elemento en una página web.

¿Qué es el modelo de cajas?

En CSS, cada elemento HTML se representa como una caja rectangular. Esa caja está compuesta por cuatro áreas concéntricas que, de adentro hacia afuera, son:

  • Content — el área donde vive el texto o la imagen.
  • Padding — espacio interior entre el contenido y el borde.
  • Border — el borde que rodea el padding.
  • Margin — espacio exterior entre el borde y los demás elementos.

El problema con box-sizing

Por defecto, cuando defines width: 300px en un elemento, ese valor corresponde únicamente al área de contenido. El padding y el border se suman al total, lo que hace que el elemento ocupe más espacio del esperado.

"Si no controlas el modelo de cajas, el modelo de cajas te controlará a ti."

— Regla no oficial del CSS

La solución es aplicar box-sizing: border-box globalmente. Con esta propiedad, el width incluye el padding y el border, haciendo los cálculos de diseño mucho más predecibles.

Ejemplo práctico

/* Buena práctica: resetear el modelo de cajas */
*,
*::before,
*::after {
  box-sizing: border-box;
}

.tarjeta {
  width: 300px;
  padding: 24px;
  border: 2px solid #ccc;
  /* Con border-box: la tarjeta mide 300px en total */
}

Márgenes y colapso

Otro comportamiento que confunde a muchos principiantes es el colapso de márgenes. Cuando dos elementos bloque adyacentes tienen márgenes verticales, estos no se suman: el mayor de los dos "gana" y el menor desaparece.

¿Cuándo ocurre el colapso?

  1. Entre elementos hermanos con márgenes verticales adyacentes.
  2. Entre un elemento padre y su primer o último hijo.
  3. En elementos vacíos que tienen margen superior e inferior.

Para evitar el colapso puedes usar padding, border o convertir el contenedor en un contexto de formato de bloque con overflow: hidden o display: flow-root.

Conclusión

El modelo de cajas no es solo teoría: cada vez que ajustes un margen, un padding o un border estarás usando este modelo. Dominarlo te ahorrará horas de depuración y te dará una base sólida para entender Flexbox, Grid y cualquier sistema de diseño.
